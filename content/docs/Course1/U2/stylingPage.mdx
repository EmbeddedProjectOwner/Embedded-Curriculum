    <>
Alright, now we're going to add a external css file that we'll use to style this.

    However, this means we're going to move all of our inline styles over to this, because it creates a cleaner, more organized code enviornment and, depending on your IDE (code editor), it'll have native syntax highlighting.
</>

/* 
This is not an offical practice/format you have to follow, however; 
I typically like to organize my css in this order:
    1. Universal selectors. (e.g. *, :root)
    2. Element-based selectors. (e.g. html, body, [descending elements...])
    3. Id Selectors (e.g. #)
    4. Classes & Keyframes (e.g. .)
    5. Media/misc (For different media screens)

And I like keeping things that are close in the DOM tree closer to one another.
*/
body {
    margin: 0;
    height: 100vh;
    display: flex;
    flex-direction: column;
}

footer {
    outline: solid;
    height: 65px;
    margin-top: auto;
    width: 100vw;
}

header {
    padding: 0.5%;
    outline: solid;
    height: 15%;
    left: 0px;
    width: 100vw;
}

.interests-container {
    padding: 2%;
    display: flex;
    flex-direction: column;
    gap: 10px
}

.interests {
    display: grid;
    padding: 2%;
    outline: solid;
    gap: 5%;

}

.grid-1 {
    grid-template-areas:
        "heading heading img"

        "img heading heading "
